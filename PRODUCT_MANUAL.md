## 1 项目概述

### 1.1 项目背景

本项目由四位同学共同开发，旨在解决教育场景中U盘文件传输的痛点。在传统教育场景中，教师和学生通常使用U盘进行文件传输，但这种方式存在诸多问题：
- **安全性低**：U盘容易丢失、被盗或感染病毒
- **效率低下**：需要物理接触设备，传输速度慢
- **兼容性差**：不同设备之间可能存在兼容性问题
- **管理困难**：难以批量管理多个设备

为了解决这些问题，我们开发了CAMFC客户端，通过蓝牙技术实现设备间的安全通信，提供更高效、更安全的文件传输方式。

### 1.2 设计理念

本项目的设计理念主要包括以下几个方面：

- **以用户为中心**：界面设计简洁直观，降低用户学习成本
- **安全性优先**：所有数据传输采用加密算法，确保信息安全
- **跨平台兼容**：支持Windows/macOS/Linux三大主流操作系统
- **轻量高效**：基于Tauri框架构建，内存占用小，启动速度快
- **可扩展性强**：采用模块化设计，便于后续功能扩展

### 1.3 开发目标

本项目的主要开发目标包括：

- 实现与Cpen蓝牙设备的安全通信
- 提供一次性密码（TOTP）获取功能
- 实现文件安全传输功能
- 提供设备管理功能
- 支持主题切换和个性化设置
- 确保跨平台兼容性和稳定性

## 2 产品介绍

### 2.1 产品简介

CAMFC（Cloud AI Multi Function Clicker）是一款专为教育场景设计的蓝牙设备管理客户端，由四位同学共同开发。该产品通过蓝牙技术实现与 Cpen 设备的安全通信，提供一次性密码（TOTP）获取、设备管理和文件传输等功能，旨在解决教育场景中U盘文件传输的痛点。

### 2.2 核心价值

- **安全可靠**：采用蓝牙5.0技术，确保数据传输的安全性和稳定性
- **跨平台支持**：支持Windows/macOS/Linux三大主流操作系统
- **轻量高效**：基于Tauri框架构建，内存占用小，启动速度快
- **用户友好**：简洁直观的界面设计，降低用户学习成本

### 2.3 应用场景

- **教育场景**：取代传统U盘，实现文件安全传输
- **设备管理**：批量管理多个Cpen蓝牙设备
- **身份验证**：通过TOTP实现二次身份验证
- **远程控制**：支持远程设备操作和管理

## 3 产品功能

### 3.1 蓝牙连接管理

#### 功能描述
- **智能扫描**：自动扫描附近的蓝牙设备，实时更新设备列表
- **设备识别**：基于设备名前缀智能识别 Cpen 设备，过滤非目标设备
- **连接管理**：自动维护单一有效连接，避免多设备冲突
- **重连机制**：设备断开后自动尝试重新连接，提升使用体验

#### 使用场景
- 首次连接Cpen设备
- 设备断开后自动重连
- 批量管理多个蓝牙设备

### 3.2 TOTP 安全验证

#### 功能描述
- **密码获取**：向连接的 Cpen 设备发送 `getTotp` 命令获取一次性密码
- **时间同步**：自动设置设备时间，确保 TOTP 生成的准确性
- **缓存优化**：30 秒缓存机制，避免频繁请求设备，延长设备电池寿命
- **安全传输**：所有数据传输采用加密算法，确保密码安全

#### 使用场景
- 身份验证场景
- 安全登录系统
- 二次验证环节

### 3.3 设备信息管理

#### 功能描述
- **唯一标识**：获取设备唯一 ID（UUID），用于设备身份验证
- **状态监控**：实时显示连接状态、信号强度等设备信息
- **手动控制**：支持手动断开/重新连接设备，满足个性化需求
- **历史记录**：记录设备连接历史，便于故障排查

#### 使用场景
- 设备身份验证
- 设备状态监控
- 故障排查

### 3.4 主题系统

#### 功能描述
- **多主题支持**：内置亮色/暗色两套主题，满足不同使用场景
- **系统跟随**：自动跟随系统主题，保持与操作系统风格一致
- **偏好记忆**：用户主题偏好自动保存到 localStorage，下次启动自动恢复
- **平滑过渡**：主题切换采用平滑动画效果，提升视觉体验

#### 使用场景
- 不同光线环境下的使用
- 个性化界面设置
- 护眼模式切换

### 3.5 文件传输

#### 功能描述
- **安全传输**：通过蓝牙技术实现文件安全传输，取代传统U盘
- **断点续传**：支持文件断点续传，避免网络中断导致传输失败
- **批量操作**：支持批量文件传输，提升工作效率
- **进度显示**：实时显示文件传输进度，让用户随时了解传输状态

#### 使用场景
- 教育场景文件传输
- 办公场景文件共享
- 跨设备数据同步

## 4 制作过程

### 4.1 开发流程

本项目的开发流程主要包括以下几个阶段：

1. **需求分析阶段**（第1-2周）

   - 制定项目计划和时间表

2. **设计阶段**（第3-4周）
   - 设计系统架构和技术方案
   - 制定UI/UX设计规范
   - 绘制界面原型图

3. **开发阶段**（第5-12周）
   - 搭建项目基础框架
   - 实现蓝牙连接管理功能
   - 实现TOTP安全验证功能
   - 实现文件传输功能
   - 实现主题系统和个性化设置

4. **测试阶段**（第13-14周）
   - 进行单元测试和集成测试
   - 进行功能测试和性能测试
   - 修复发现的bug和问题

5. **发布阶段**（第15周）
   - 构建生产版本
   - 编写产品文档和使用手册
   - 发布项目到GitHub

### 4.2 技术选型

#### 前端技术
- **Vue 3**：前端框架，采用组合式API提高代码复用性
- **TypeScript**：类型系统，增强代码可维护性
- **Pinia**：状态管理，用于管理蓝牙连接状态
- **Vue Router**：路由管理，实现页面导航
- **Vite**：构建工具，提供极速热更新和打包速度

#### 后端技术
- **Tauri**：桌面应用框架，轻量高效，跨平台支持
- **Rust**：系统编程语言，内存安全，性能优异
- **Tokio**：异步运行时，用于处理蓝牙异步操作
- **btleplug**：蓝牙通信库，跨平台蓝牙支持

#### 样式技术
- **CSS 变量**：实现主题切换和个性化设置
- **Normalize.css**：重置浏览器默认样式
- **Remix Icon**：图标库，提供丰富的图标资源

### 4.3 遇到的挑战

在开发过程中，我们遇到了以下主要挑战：

1. **蓝牙通信问题**
   - 不同平台蓝牙API的差异
   - 设备连接不稳定
   - 数据传输效率低

2. **跨平台兼容性问题**
   - Windows/macOS/Linux系统差异
   - 不同浏览器的兼容性问题
   - 蓝牙适配器的兼容性问题

3. **性能优化问题**
   - 前端渲染性能优化
   - 蓝牙通信性能优化
   - 内存占用优化

### 4.4 解决方案

针对上述挑战，我们采取了以下解决方案：

1. **蓝牙通信问题**
   - 使用btleplug库统一蓝牙API调用
   - 实现自动重连机制
   - 优化数据传输协议

2. **跨平台兼容性问题**
   - 使用Tauri框架封装系统差异
   - 进行多平台测试和适配
   - 提供蓝牙适配器兼容性列表

3. **性能优化问题**
   - 使用Vue 3的响应式系统优化渲染性能
   - 实现数据缓存机制
   - 优化内存使用和资源管理

## 6 安装指南

### 6.1 环境要求

#### 必选依赖
- Node.js 18.0+ 或 20.0+（推荐使用 LTS 版本）
- Rust 1.70.0+（Tauri 框架依赖）
- Git（版本控制和依赖拉取）

#### 系统特定依赖

##### Windows
- Windows 10 1903+ 或 Windows 11
- Visual Studio Build Tools（安装时选择"Desktop development with C++"）
- 蓝牙适配器（支持蓝牙 4.0+）

##### macOS
- macOS 11.0+（Big Sur 或更高版本）
- Xcode 13.0+（安装后需运行 `xcode-select --install` 安装命令行工具）
- 蓝牙适配器（内置或外接）

##### Linux
- Ubuntu 20.04+ 或其他主流发行版
- WebKit2GTK 开发库
- 蓝牙开发库（libbluetooth-dev）

### 6.2 安装步骤

#### 开发模式安装
```bash
# 1. 克隆项目
git clone <repository-url>
cd CAMFC-client

# 2. 安装前端依赖
npm install

# 3. 启动开发服务器
npm run dev

# 4. 启动 Tauri 窗口
npm run tauri dev
```

#### 生产模式安装
```bash
# 1. 构建前端
npm run build

# 2. 构建桌面应用
npm run tauri build
```

### 6.3 常见问题

- **依赖安装失败**：尝试删除 `node_modules` 和 `package-lock.json` 后重新安装
- **Tauri 启动失败**：检查 Rust 版本是否符合要求，或运行 `rustup update` 更新
- **蓝牙无法扫描**：确保蓝牙适配器已启用，且设备处于可发现状态

## 7 使用指南

### 7.1 快速上手

1. 打开 CAMFC 客户端
2. 点击"扫描设备"按钮
3. 选择要连接的 Cpen 设备
4. 等待连接成功
5. 开始使用各项功能

### 7.2 蓝牙连接

1. 确保设备蓝牙已开启
2. 点击客户端首页的"扫描设备"按钮
3. 从设备列表中选择要连接的 Cpen 设备
4. 等待连接成功（连接状态会显示为"已连接"）
5. 若连接失败，可点击"重新连接"按钮重试

### 7.3 TOTP 获取

1. 确保已成功连接 Cpen 设备
2. 点击"获取 TOTP"按钮
3. 等待系统返回一次性密码
4. 复制密码用于身份验证
5. 密码会自动缓存 30 秒，期间无需重复请求

### 7.4 文件传输

1. 确保已成功连接 Cpen 设备
2. 点击"文件传输"按钮
3. 选择要传输的文件
4. 点击"开始传输"按钮
5. 等待传输完成（可查看实时传输进度）
6. 若传输中断，可点击"继续传输"按钮恢复

### 7.5 主题切换

1. 点击客户端右上角的"设置"按钮
2. 选择"主题设置"
3. 选择"亮色主题"或"暗色主题"
4. 系统会自动应用所选主题
5. 下次启动时会自动保留上次选择的主题

## 9 常见问题

### 9.1 连接问题

**Q: 无法扫描到蓝牙设备？**
A: 请确保设备蓝牙已开启，且设备处于可发现状态。若仍无法扫描，可尝试重启蓝牙适配器或设备。

**Q: 连接蓝牙设备失败？**
A: 请检查设备电量是否充足，蓝牙信号是否稳定。若仍无法连接，可尝试重启设备或客户端。

**Q: 蓝牙连接频繁断开？**
A: 请确保设备之间的距离在有效范围内（10 米以内），避免障碍物遮挡。若仍频繁断开，可尝试更换蓝牙适配器。

### 9.2 功能问题

**Q: 无法获取 TOTP 密码？**
A: 请确保已成功连接 Cpen 设备，且设备时间已同步。若仍无法获取，可尝试重新连接设备。

**Q: 文件传输速度慢？**
A: 请确保设备之间的距离在有效范围内，蓝牙信号稳定。若仍速度慢，可尝试减少同时传输的文件数量。

**Q: 主题切换不生效？**
A: 请确保客户端版本为最新版本。若仍不生效，可尝试重启客户端。

### 9.3 性能问题

**Q: 客户端启动速度慢？**
A: 请检查系统资源是否充足，关闭不必要的后台程序。若仍启动慢，可尝试重新安装客户端。

**Q: 客户端占用内存高？**
A: 请确保客户端版本为最新版本。若仍占用高，可尝试关闭不必要的功能模块。
